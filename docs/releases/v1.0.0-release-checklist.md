# v1.0.0 Release Checklist

Status: Release-prep checklist. Completing this document does not create a
tag, publish a release, publish a package, or certify production readiness.

This checklist covers the reviewed release candidate prepared by issue
[#100](https://github.com/ElvinMorales/agent-librarian/issues/100). The
maintainer retains the final review, tag, and GitHub release actions.

## Repository State

- [ ] Branch is based on latest `main`.
- [ ] Working tree is clean before final PR review.
- [ ] PR is reviewed and merged.
- [ ] Maintainer confirms no unexpected generated files are committed.

## Version and Changelog

- [ ] Version is `1.0.0` in authoritative package metadata.
- [ ] `agent-librarian --version` reports `agent-librarian 1.0.0`.
- [ ] `CHANGELOG.md` has a `1.0.0 - 2026-06-29` section.
- [ ] `Unreleased` is reset for future work.

## Validation

- [ ] Full test suite passes.
- [ ] Package conformance passes.
- [ ] Source snapshot conformance passes.
- [ ] Synthetic SharePoint-style demo commands pass.
- [ ] `git diff --check` passes.

Run validation with repository-local temporary directories:

```powershell
Remove-Item -Recurse -Force .tmp\pytest-basetemp, .tmp\pytest-cache -ErrorAction SilentlyContinue
New-Item -ItemType Directory -Force .tmp\pytest-basetemp, .tmp\pytest-cache | Out-Null
$env:TMP = "$PWD\.tmp"
$env:TEMP = "$PWD\.tmp"

python -m pytest --basetemp=.tmp\pytest-basetemp -o cache_dir=.tmp\pytest-cache
python scripts/check_packages.py
python scripts/check_source_snapshot.py examples/source-snapshots/synthetic-team-space
agent-librarian catalog examples/source-snapshots/synthetic-team-space/files --out .tmp\synthetic-sharepoint-catalog --strict
agent-librarian validate .tmp\synthetic-sharepoint-catalog
agent-librarian report .tmp\synthetic-sharepoint-catalog
agent-librarian present .tmp\synthetic-sharepoint-catalog --out .tmp\synthetic-sharepoint-presentation
agent-librarian --version
git diff --check
```

This repository does not expose `python -m agent_librarian`; the documented
and required version interface is the `agent-librarian` console script. Do not
add a module entry point solely for this release.

## Compatibility Contract Confirmation

The release-prep candidate maps the
[v1 readiness and compatibility contract](v1-readiness-and-compatibility.md)
to these repository surfaces:

| Contract area | Candidate evidence |
| --- | --- |
| Deterministic CLI | `src/agent_librarian/cli.py` exposes local `catalog`, `validate`, `report`, and default offline `present` commands. |
| Generated outputs and schemas | `src/agent_librarian/renderers.py`, `src/agent_librarian/schema_data/`, and validation tests define and check generated artifacts. |
| Package adapters | `agent/`, `packages/`, and `scripts/check_packages.py` define canonical and provider-mapped package surfaces and conformance. |
| Presentation layer | `src/agent_librarian/presenter.py` renders deterministic offline HTML by default; narration remains explicit and optional. |
| Source snapshot contract | `docs/adapters/source-snapshot-contract.md`, the bundled source-manifest schema, and `scripts/check_source_snapshot.py` define and check local snapshot inputs. |
| Microsoft 365 / SharePoint proof path | The boundary decision, companion adapter reference, and synthetic SharePoint-style demo provide the documented proof path without a live connector. |
| Evals and conformance | The test suite and package/source-snapshot checkers cover the bounded automated gates. |
| Public/private safety | Public-safety guidance, synthetic fixtures, sensitivity inheritance, and the review below preserve the documented boundary. |

This mapping confirms the candidate's documented surfaces; it does not claim
that a release has occurred or certify safety, correctness, completeness,
compliance, approval, publication readiness, or production readiness.

## Public-Safety Review

- [ ] No secrets, credentials, tokens, API keys, or `.env` files are committed.
- [ ] No real tenant IDs, SharePoint URLs, screenshots, private source exports,
  or employer-specific examples are committed.
- [ ] No private source snapshots or generated private outputs are committed.
- [ ] README and release notes do not imply certification, production
  readiness, private-data safety, employer endorsement, or autonomous
  publication behavior.
- [ ] No live connector, Microsoft Graph, OAuth/MSAL, provider SDK, or network
  behavior is added.

## Manual Release Actions After Merge

- [ ] Maintainer reviews the final merged commit.
- [ ] Maintainer creates the `v1.0.0` tag manually.
- [ ] Maintainer drafts the GitHub release manually.
- [ ] Maintainer confirms release language matches public-safety boundaries.
- [ ] Maintainer publishes the release only after review.

Package registry publication is outside this release-prep scope and requires
separate maintainer authorization.
